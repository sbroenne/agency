How it works

A lighter path than a dashboard

  1. Browse first. Start with the published squads so you can see the shape, mission, and source repo before adopting anything.
  2. Run Squad in your repo. Use the Bradygaster Squad workflow when you are ready to bring a team into your own codebase.
  3. Submit what works. When your squad is reusable, open a pull request so it can be validated and listed here.

Contribute

Publish through the existing GitHub flow

This registry stays static on purpose: squads live in the repository, validation runs in CI, and GitHub Pages publishes the result. No custom app routing, no extra directory mechanics.
